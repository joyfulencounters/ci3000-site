# CI3000 网站说明

项目类型：
ASP.NET WebForms 网站

开发目标：
优化 UI 风格，使其更简洁清新。

开发规则：

1、不要修改任何 C#代码
2、不要修改 .aspx.cs 文件
3、不要删除 ASP.NET服务器控件
4、不要修改数据绑定逻辑
5、只允许修改 HTML 和 CSS
6、不要改变数据库调用
7、每次只修改一个模块
8、不要给页面主容器设置 height:1% 或 overflow:hidden
9、不要通过压缩高度解决布局问题
10、修改 `word.aspx` 时，只做我指定的局部改动。
11、不要改变 `word.aspx` 的原始文件编码。
12、保存文件时必须保持与项目原文件一致的编码，优先保持为 GB2312 或 GBK，不要自动转成 UTF-8 或 UTF-8 with BOM。


UI风格：

参考：
豆瓣 + 古典文学网站

风格要求：

- 简洁
- 留白
- 中文阅读友好
- 卡片式布局

特别说明：

- 每次修改完毕后，都要把修改过的文件列出。
- 修改其他模块或者页面的时候，不要影响已有的模块或页面
- 不要改写现有中文静态文本
- 不要修改文件编码
- 只改 CSS，不重写 HTML 文案
- 如果必须改 HTML，只增删标签，不改中文内容本身
